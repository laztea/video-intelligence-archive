## 1. 최초 요청 사양 (원안)

처음 받은 스펙은 다음과 같다.

| 항목 | 내용 |
|---|---|
| **형태** | 웹서비스 — 브라우저에서 영상 업로드(진행 상황 프로그래스 바) |
| **AI 모델** | 분석·검색 모두 OpenAI GPT-5.5 — 자막 교정 · 장면 분석(Vision) · 금칙 검수(Vision) · 자연어 검색 쿼리 분해. 임베딩은 `text-embedding-3-small` |
| **입력** | 영상 파일(업로드, 1개 이상 누적 가능) |
| **출력** | 검색 가능한 영속 아카이브 · 운영 절차서(.md) · 텔레그램 알림 |
| **저장** | 이중 저장소 — ① SQLite(테이블 + FTS5 전문검색) ② ChromaDB(벡터, 영속) |
| **검색** | 하이브리드 = 키워드(FTS5) + 의미(임베딩) 점수 합산. 모드 4종: `hybrid` / `keyword` / `vector` / `filter` |
| **핵심 차별점** | 검색 결과마다 **출처(keyword/vector/both)** 와 **왜 나왔는지** 설명 반환 |
| **추가** | 텔레그램 봇으로 진행 결과 알림 |

**제약 4가지**
1. 결과는 재시작해도 유지(파일 영속)
2. API 키·토큰은 `.env`
3. 한 단계 실패해도 가능한 범위까지 색인
4. GPT/임베딩 호출 키는 사용 시점에 지연 초기화

> 원안은 STT(음성→텍스트) 방법이 비어 있었고, 브레인스토밍에서 **Whisper STT**로 확정했다.
> (자막 교정의 입력 = Whisper 결과)

## 2. 원안대로 구현된 것

- 멀티모달 파이프라인: `persist → audio → transcribe(Whisper) → subtitle(GPT) → keyframes(ffmpeg) → vision(GPT) → embed → index-sql → index-vec → finalize`
- 이중 저장소(SQLite+FTS5 / ChromaDB), 4모드 하이브리드 검색, 출처·이유 설명
- 단계별 체크포인트·부분 색인·재시작 재개(제약①③), `.env` 키(제약②), 지연 초기화(제약④)
- 텔레그램 알림, 운영 절차서(`docs/RUNBOOK.md`) 자동 생성
- 모듈 분해: `config·events·media·transcribe·embed·enrich·store·pipeline·search·notify·web·runbook`

설계/계획 문서: `docs/superpowers/specs/2026-06-11-…-design.md`, `docs/superpowers/plans/2026-06-11-…md`

## 3. 구현 후 추가·변경된 사항 (세션 중 요청·발견으로 진화)

### 3.1 텔레그램 알림
- **단계별 알림 → 완료 1건만** (스팸 방지 요청)
- 완료 메시지를 **요약형**으로: 색인한 자막/장면 수, 금칙 종류별 건수, 실패 단계 표시
  - 예) `✅ 「파일명」 처리 완료 / • 색인: 자막 66개 · 장면 69개 / • ⚑ 금칙 40건 (logo 30, violence 8, sexual 2)`

### 3.2 UI/UX 전면 강화 (원안은 "업로드만")
- 다크 "필름 도시에" 테마(Fraunces/Spline Sans/JetBrains Mono)
- **아카이브 그리드**(썸네일·상태·금칙·카운트), **드래그&드롭 멀티 업로드** + SSE 진행바 + 토스트
- **영상 상세**: Range 스트리밍 플레이어 + 클릭 가능한 자막/장면 타임라인(시킹)
- **설명 가능한 검색 결과 카드**: 출처 배지 + WHY + **장면 썸네일**, 클릭 시 상세에서 해당 구간 **하이라이트**
- 헤더 레이아웃 수정: 검색을 독립 행으로 분리(업로드 버튼 오클릭 방지)
- 신규 백엔드: `/video/{id}`, `/media/{id}`(seek), `/thumb/{id}`, `/frame/{chunk_id}`

### 3.3 검색 품질 개선 (실사용 피드백 기반)
- **한국어 FTS**: 기본 토크나이저는 부분일치 불가 → **trigram** 도입
- **2글자 한국어**: trigram이 3글자 미만 미색인 → **LIKE 부분일치 보완**
- **FTS 안전화**: 쉼표 등 특수문자 쿼리에서 500 에러 → 토큰화·인용 처리
- **coverage 랭킹**: 다중 키워드("여자 사무실")는 **매칭 키워드 수 우선** 정렬(둘 다 포함이 위로)
- **hybrid 키워드 분리**: 벡터는 문장, 키워드는 **추출 키워드** 사용(불용어 매칭 버그 제거)
- **유의어 확장**: 쿼리 분해가 동의어를 키워드로 확장(`오피스→사무실, 직장, 회사…`)
- 설명에 **실제 매칭어**만 표기(이전엔 본문 전체 단어 나열 버그)

### 3.4 DB 뷰 페이지 (신규 요청)
- 관계형 스키마(테이블·컬럼·PK), 논리적 FK 관계도, ChromaDB 벡터 연결, **실시간 테이블 데이터** 열람
- 백엔드: `/db/schema`, `/db/table/{name}`(화이트리스트)

### 3.5 검색 품질 자동 평가 파이프라인 (신규 요청)
- `eval/run_eval.py` — 사람 개입 없이 끝까지 실행
- **known-item 설계**: 장면 설명에서 LLM이 골드 쿼리 100개 생성(정답=출처 장면)
- 지표: **Recall@5 · Precision@5(LLM-as-judge) · MRR**, 실패 패턴 분석, 자체 완결 HTML 리포트
- 앱 `평가 ↗` 탭(`/eval`)으로 열람
- **검증 결과(N=100): Recall@5 87.0% · Precision@5 43.8% · MRR 0.650 · miss 13**
  - (스모크 N=4의 100%는 표본 과소 + 방법론적 낙관 편향이었음 — 아래 참고)

### 3.6 장면 이해 — 대사 맥락 융합 (신규 요청, "학교/사무실 혼동" 해결)
- **원인**: 장면이 keyframe 1장 단위라 클로즈업은 "실내"로만 묘사 → 장소 단서 부재
- **해결**: Vision 호출에 **그 시점 ±6초 대사**를 함께 투입 → 모호한 장소·상황·관계를 대사로 추론(근거 없으면 보류)
- keyframe에 ffmpeg `showinfo` **실제 타임스탬프** 부여(대사 정렬 + UI 시킹 정확도 개선)
- 적용 시점: **새로 색인되는 영상부터** (기존 영상은 재처리 필요)

### 3.7 안정성/버그 수정 (실사용·QA 중 발견)
- 서버 기동 실패: `python-multipart` 누락 → 추가
- keyframe ffmpeg가 장면전환 없는 영상에서 항상 실패 → **첫 프레임 항상 포함**
- scene 청크 적재 `KeyError('start')` → `start_s/end_s`로 수정
- **동시 요청 시 SQLite segfault**(단일 커넥션 공유) → **스레드별 커넥션 + WAL**

### 3.8 배포
- GitHub 공개 저장소 + README(스크린샷 포함) — https://github.com/laztea/video-intelligence-archive
- `.env`·`data/`·eval 생성물은 비공개(gitignore)

## 4. 알려진 한계 / 다음 버전에서 다룰 거리

1. **keyword 단독 모드** — 한글 2글자 미만·유의어 약함(hybrid/vector가 보완). 형태소 분석기 도입 검토.
2. **Whisper 환각 자막** — 무음/배경음 구간에서 `ご視聴ありがとうございました` 등 환각 발생 → 대사 맥락 오염. **환각 필터** 필요.
3. **평가의 낙관 편향** — 쿼리를 장면 설명에서 생성하면 임베딩이 쉽게 찾아 점수 부풀려짐. **적대적 쿼리 모드**(모호·유의어 변형)로 현실 난이도 측정.
4. **장면 단위의 맥락 부재** — 프레임 독립 분석. 영상 레벨 요약/태그, 인접 프레임 병합, 멀티프레임 Vision 등으로 확장 여지.
5. **GPT-5.5 모델명** — 코드에 스펙대로 하드코딩. 실제 모델명과 다르면 `enrich/`·`search/query.py`의 `MODEL` 상수만 교체.
6. **재처리 흐름 부재** — 코드 개선(대사 맥락 등) 후 기존 영상에 소급 적용하려면 수동. **재색인 트리거** 기능 검토.
